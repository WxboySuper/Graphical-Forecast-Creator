# v1.4.0 Phase 3 PRD: Stripe Billing and Entitlements

**Execution target:** one implementation branch and one PR

## Objective
Introduce premium subscription infrastructure that cleanly gates hosted cloud-storage features while keeping all core local GFC functionality free.

## Why Now
Billing needs to be stable before premium cloud storage ships. This phase turns the product boundary into a working entitlement system so the later storage phase can focus on UX and data safety instead of payment plumbing.

## In Scope
- Pricing page or pricing surface
- Stripe checkout flow
- Stripe billing portal flow
- Subscription entitlement state mirrored into app-readable data
- Launch promo and standard pricing configuration
- Closed-beta override path for invited users

## Out of Scope
- Cloud cycle library CRUD
- User metrics UI
- Admin charts beyond the minimum needed to inspect entitlements
- Additional premium perks beyond hosted storage

## Implementation Checklist
- [ ] Define Stripe product and price IDs for monthly, launch-promo annual, and standard annual plans
- [ ] Add `server/` endpoints for checkout session creation and billing portal session creation
- [ ] Add a Stripe webhook handler that updates subscription and entitlement records after checkout, renewal, cancellation, and payment state changes
- [ ] Create one client entitlement source of truth that the UI can read without calling Stripe directly
- [ ] Gate premium UI on entitlement state instead of on raw auth state
- [ ] Add a closed-beta allowlist or override path for invited testers
- [ ] Expose the launch promo window as config so pricing can switch from `$25/year` to `$30/year` without a code rewrite
- [ ] Add pricing and account messaging that clearly explains premium as hosted sync/storage, not core forecasting access
- [ ] Define downgrade behavior for v1.4.0: premium expiration disables new cloud saves and background sync, but does not remove local access to existing work

## Test Checklist
- [ ] Monthly checkout creates the expected entitlement
- [ ] Annual checkout applies the correct configured price
- [ ] Billing portal opens for an existing subscriber
- [ ] Webhooks correctly update entitlement state for subscription creation, cancellation, and renewal changes
- [ ] Non-premium users never see cloud-save actions enabled
- [ ] Beta override users can access premium-gated flows without live billing
- [ ] Pricing surfaces fall back safely when Stripe is not configured

## Rollout Notes
- Run this phase on `beta` with tester-only access before public pricing appears
- Billing copy should stay simple and repeat the free-vs-premium boundary everywhere the user can pay
- Keep provider dashboards as the deeper operational truth even after a minimal admin UI exists

## Dependencies / Blockers
- Depends on Phase 2 accounts
- Requires working Stripe products, price IDs, and webhook secrets
- Premium cloud storage work should not start until entitlement propagation is reliable
