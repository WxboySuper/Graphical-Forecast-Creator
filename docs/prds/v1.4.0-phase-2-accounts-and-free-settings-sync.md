# v1.4.0 Phase 2 PRD: Accounts and Free Settings Sync

**Execution target:** one implementation branch and one PR

## Objective
Add optional user accounts and synced profile/settings so users can sign in across devices without changing the local-first nature of GFC.

## Why Now
Identity is the prerequisite for both premium entitlements and user-facing progress metrics. Shipping the account layer early also gives beta users time to validate the sign-in experience before cloud storage and billing are layered on.

## In Scope
- Firebase Auth setup
- Google, GitHub, and email/password sign-in
- Signed-in vs signed-out app state
- Navbar and home-page account entry points
- Account page/profile shell
- Free synced profile and settings data
- Safe merge behavior between local preferences and cloud settings

## Out of Scope
- Stripe billing
- Premium cloud cycle storage
- Admin dashboard
- User-facing verification score UI

## Implementation Checklist
- [ ] Add Firebase client initialization with hosted feature guards
- [ ] Create auth state management for loading, signed-out, signed-in, and error states
- [ ] Add sign-in and sign-out entry points in the existing navigation without blocking local usage
- [ ] Support Google, GitHub, and email/password providers
- [ ] Create a user profile document on first successful sign-in
- [ ] Create a synced settings document for dark mode, base map style, overlay preferences, and default forecaster/profile fields
- [ ] Define first-load merge behavior: prefer cloud settings when present, otherwise seed cloud from the current local values
- [ ] Add `/account` with profile basics, linked-provider visibility, and a placeholder plan/metrics section
- [ ] Keep current signed-out flows fully functional, including local auto-save, local history, JSON import/export, and verification
- [ ] Add feature-flagged messaging that encourages sign-in without making it feel mandatory

## Test Checklist
- [ ] Google sign-in flow works in hosted mode
- [ ] GitHub sign-in flow works in hosted mode
- [ ] Email/password sign-up, sign-in, and sign-out work in hosted mode
- [ ] Signed-out users can still use the app exactly as before
- [ ] Dark mode and base map preferences sync across two sessions for the same account
- [ ] Local preferences are not lost if the cloud settings document is missing or temporarily unavailable
- [ ] Account route and navigation fail gracefully when hosted auth is disabled

## Rollout Notes
- Enable providers individually if needed during internal testing
- Closed beta users should start here even before premium is turned on
- Support copy should emphasize convenience and cross-device continuity, not account lock-in

## Dependencies / Blockers
- Depends on Phase 1 env/config and privacy groundwork
- Needs Firebase project configuration and enabled providers
- Metrics and billing work should wait until auth state is stable enough for beta users
