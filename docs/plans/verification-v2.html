Verification v2 — Forecast Grade

Formula version gfc-ver-1 · gate verificationRelaunch · tracker #430

Relaunch Verification as a map-first evidence dashboard with a versioned Forecast Grade, explicit sources, grade history, and one shareable grade-plus-map card — behind verificationRelaunch, without removing classic Verification. Primary job: learn fast whether the forecast captured what happened. Audience: forecasters learning the craft — clear for beginners, not bogged down for experts. Progressive disclosure via labeled expandable sections, never a Basic/Advanced switch, and no coaching / AI prose.

1. Coexistence & feature gate

  • Classic /verification (VerificationMode) stays the live default when the flag is off.
  • V2 mounts only when verificationRelaunch is exposed. Dogfood: exposure.local: true only; beta/staging/production stay false.
  • Disabled-side-effect tests ensure a flag-off build cannot boot the v2 dashboard or gfc-ver-1 engine as a side effect (lazy route + side-effect module registry).
  • Deleting classic Verification is out of scope for this effort.

2. Sources & history (capability-first)

TierPackage inputHistory

Source selection is always explicit — there is no automatic handoff from the Forecast Editor. Completed runs auto-save an immutable snapshot (formula version + source snapshot); re-runs create new results and never rewrite history.

3. UI / dashboard

Layout

  • Desktop: equal-weight map and results dashboard; the map is the evidence surface.
  • Mobile: map-first with a compact Forecast Grade overlay/drawer; details reachable without overflow.
  • Landscape phone: keep the mobile interaction model when height is constrained.
  • Map controls always reachable: hazard, day, evidence.
  • Selecting a grade component/product emphasizes related geometry on the map.
  • Selectable report table is the keyboard/screen-reader equivalent of map highlights; the map stays primary.

Result view

  • Headline: Forecast Grade (0–100 + letter, one decimal) + component scores.
  • Expandable sections titled exactly Score breakdown and Data quality.
  • Data quality: Good / Limited / Blocked; quiet day says No reports.
  • Missing products: Not evaluated row with classic warning treatment (no coaching).
  • History trend filterable by hazard. Comparison mode is out of scope.

Run UX

  • Accuracy over a fixed latency budget; long runs show staged foreground progress and complete automatically.
  • Invalid inputs block the run. Sparse-but-valid may withhold the package grade while showing components (Limited).

Share

  • One anonymous grade-plus-map card (map-led). Offer download, native share (when available), and copy (when supported).
  • No identity by default.

Methodology

Public methodology page linked from the dashboard documents the formulas, caveats, version history, SPC 25-mile citation, and intent/yield rationale. The formula version is embedded in the UI, docs, and exports.

4. Math gfc-ver-1

Two layers: the SPC occurrence definition (25-mile neighborhood + Armchair-style area buffers / per-tier contingency) and the GFC intent layer (event yield/concentration + severity). No distance-from-polygon decay inside the halo.

ComponentWeightDefinition
Probability skill25%Spatial Brier / BSS over a ~10 km grid; higher-f empty cells hurt more; quiet overforecast still scores.
Spatial contingency25%Area hit/miss/FA CSI per probability tier, combined across tiers.
False-alarm discipline15%Probability-weighted areal FAR.
Event yield25%expected = baseline(f) + area × density(f); yield = min(1, reports/expected). Fails huge 30%+1; softens tiny 45%+1.
Severity10%Sig reports vs sig contours in the 25-mi neighborhood; N/A if neither; soft ~70 penalty if sig drawn and none observed.

Package rollup: equal weight among present products (categorical / tornado / wind / hail); absent → Not evaluated. Data quality Blocked / Limited / Good as above. Geometry quality is a gate, not a weight. Letter bands: A ≥ 90, B ≥ 80, C ≥ 70, D ≥ 60, F < 60. Categorical → probability: TSTM 0.05, MRGL 0.10, SLGT 0.15, ENH 0.30, MDT 0.40, HIGH 0.45.

Reference inspiration (area method, not gamed for contests): Armchair Forecaster — Verification. See also the in-app Forecast Grade methodology page.

5. Architecture

  1. Pure engine (no React): neighborhood/buffers, area metrics, Brier, yield, severity, composite, snapshots — src/utils/verificationV2/.
  2. Source adapters: file + premium cloud; SPC reports; validate or block — sources.ts.
  3. Snapshot store: auto-save; free grade cards; premium full restore — gradeHistory.ts.
  4. Feature-gated UI: V2 shell on /verification when flag on; else classic — src/components/ForecastGrade/.
  5. Share + docs: PNG card; methodology page for gfc-ver-1.
