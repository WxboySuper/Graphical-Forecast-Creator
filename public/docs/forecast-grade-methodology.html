Forecast Grade Methodology

Formula version gfc-ver-1 · tracker #430

The Forecast Grade measures how well a severe-weather outlook captured what actually happened. It combines the SPC occurrence definition with an intent layer that rewards well-calibrated, concentrated high-probability forecasts. Scores and metrics speak for themselves — the dashboard never adds coaching prose.

SPC spatial contract

Probabilities are the chance of severe weather within 25 miles of any point in a contour (SPC "About the Convective Outlooks"). Each storm report is buffered by 25 miles (~40 km). Where contours are nested, the effective claim at a location is the highest probability whose neighborhood reaches it. There is no distance-from-polygon decay inside the 25-mile halo — a location is either within the neighborhood or it is not.

One report only supports the area within ~25 miles of that report; a single report does not verify an entire large probability blob.

Composite components (per product)

Weights are dropped and renormalized when a component is Not evaluated.

ComponentWeightWhat it measures
Probability skill25%Spatial Brier Skill Score over a ~10 km grid of the forecast envelope + 25 mi buffer. Higher-probability empty cells hurt more; quiet overforecast days still score.
Spatial contingency25%Armchair-style area hit/miss/false-alarm CSI computed per probability tier, then combined across tiers.
False-alarm discipline15%Probability-weighted areal FAR, so empty high-probability paint is penalized more than an empty 2% skirt.
Event yield / concentration25%Intent layer (see below).
Severity10%Significant contours vs significant reports in the 25-mi neighborhood. Not evaluated when neither exists; soft ~70 penalty when sig is drawn but nothing significant verifies.

Event yield / concentration (intent)

For each drawn core (f ≥ 0.15, 0.30, 0.45): expected = baseline(f) + area(core) × density_factor(f) and yield = min(1, reports_near(core) / max(expected, ε)). Higher probability implies a more concentrated claim, so the density factor rises with f. This makes a huge 30% core with a single report fail on yield while softening a tiny 45% core with a single report — neither is a free "full verify". Present cores are averaged.

Package rollup & data quality

Present severe-hazard products (tornado, wind, hail) are combined with equal weight; absent products are marked Not evaluated. The categorical outlook and TSTM areas are display-only composite layers — they are not scored. Verification requires drawn severe-hazard probability contours. Data quality is a gate, not a score weight:

StatusMeaning
GoodEnough geometry and reports to trust every applicable component (including a clean quiet-day overforecast path labeled No reports).
LimitedSparse non-quiet reports; components are shown but the package grade is withheld.
BlockedMissing forecast geometry or unreadable reports — no run.

Letter bands

A ≥ 90, B ≥ 80, C ≥ 70, D ≥ 60, F < 60. Grades display one decimal (e.g. 82.4 / B).

Display vs graded layers

The dashboard defaults to the categorical map for context, but grades only the tornado, wind, and hail probability contours forecasters draw. Categorical risk labels and TSTM (sub-severe general thunder) are composite or manual layers without a standalone probabilistic field, so they are excluded from the Forecast Grade.

Caveats

  • Grades are an educational tool, not an official verification product.
  • Official grades change only via versioned GFC releases; there are no per-user weight sliders.
  • Hit/miss diagnostics are informational and do not by themselves determine the grade.
